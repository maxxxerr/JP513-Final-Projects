* **Дедлайн:** Работы принимаются до **13 июля включительно**. Пул-реквесты, отправленные позже, рассматриваться не будут.

## Инструкция по сдаче проекта (Pull Request)

Чтобы загрузить свой проект на проверку, строго следуйте этим шагам:

### Шаг 1. Создание копии (Fork)
В правом верхнем углу этой страницы найдите и нажмите кнопку **Fork** (затем **Create fork**). Это создаст копию репозитория в вашем личном аккаунте GitHub.

### Шаг 2. Клонирование репозитория
Откройте терминал/консоль на компьютере и склонируйте **свой форкнутый** репозиторий (замените `ВАШ_НИКНЕЙМ` на ваше имя пользователя GitHub):

```bash
git clone https://github.com/ВАШ_НИКНЕЙМ/JP513-Final-Projects.git
cd JP513-Final-Projects
```

### Шаг 3. Создание новой ветки

Обязательно создайте отдельную ветку для своей работы. Назовите её своей фамилией (на латинице):

```bash
git checkout -b feature-ivanov
```

### Шаг 4. Размещение проекта

1. Перейдите в папку `students/`.
2. Создайте внутри неё свою персональную папку в формате `фамилия_имя` (например: `bogdanov_ivan`).
3. Перенесите **все файлы вашего проекта** в созданную папку.
4. Позаботьтесь о файле `.gitignore` в корне вашего проекта, чтобы случайно не отправить виртуальное окружение (`.venv`, `venv`), кэш (`__pycache__`) или локальные базы данных.

### Шаг 5. Отправка кода на GitHub

Зафиксируйте изменения коммитом и отправьте ветку в свой удаленный репозиторий:

```bash
git add .
git commit -m "Добавлен финальный проект: Богданов Иван"
git push origin feature-ivanov
```

### Шаг 6. Создание Pull Request

1. Зайдите на страницу **своего** репозитория на сайте GitHub.
2. Сверху появится желтая плашка с кнопкой **Compare & pull request**. Нажмите на неё.
3. В заголовке (Title) укажите ваши настоящие ФИО: *«Финальный проект: Богданов Иван»*.
4. Нажмите синюю кнопку **Create pull request**.

## Важные правила

1. **Не трогайте чужое!** Вам разрешено создавать и изменять файлы **только** внутри вашей личной папки в директории `students/`. Если ваш Пул-реквеств затронет файлы ТЗ или папки других студентов, он будет отклонен.
2. Внутри вашей папки крайне рекомендуется создать файл `README.md` с краткой инструкцией по запуску вашего Python-скрипта или приложения (какие библиотеки установить, какой файл главный).
